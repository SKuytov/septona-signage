# Septona Lobby Signage — Седмичен график

Уеб приложение за табло в лобито, което показва седмичния график на работниците от качен `.xlsx` файл, като запазва цвета на всеки работник според неговата **автобусна линия**. Проектирано за 65" iiyama дисплей (1080p / 4K).

A lobby signage web app that displays the weekly worker schedule from an uploaded `.xlsx`, preserving each worker's **bus-line background color**. Built for a 65" iiyama signage display.

---

## Функции / Features

- **Автоматично разчитане на .xlsx** — имена, локации, смени и цветове на автобусните линии (резолюция на theme цветове + tint).
- **Две оформления / Two layouts:**
  - **Табло (Board)** — тъмен dashboard в стил на екрана, номерът на всеки работник е оцветен по линия.
  - **Графика (Grid)** — точен цветен грид като PDF-а, всеки ред с фона на линията.
- **Качване през уеб** — `/admin` страница с drag-drop за качване на новия седмичен файл.
- **Автоматично обновяване** — таблото проверява за нов график на всеки 20 сек.
- **Kiosk режим** — `?kiosk=1` скрива контролите; `?rotate=20` върти секциите на всеки 20 сек.
- **Часовник на живо** (Europe/Sofia) и осветяване на текущата смяна.
- **Съобщения върху таблото / On-screen messages** — публикувай съобщения (със или без изображение) в три режима:
  - **Бягаща лента (banner)** — тънка лента долу върху графика.
  - **Цял екран (slide)** — ротира се на пълен екран между графика.
  - **Спешно (takeover)** — червено пулсиращо известие, което заема целия екран и прекъсва всичко.
  - Всяко съобщение има приоритет/цвят (Информация / Внимание / Спешно), по избор начало/край на показване и времетраене. Управляват се от `/messages.html`.
- **Android kiosk приложение** — готов `.apk` (WebView обвивка) за инсталиране на дисплея; авто-старт, екранът не заспива, устойчив на офлайн. Виж [`android/`](./android/README.md).

## Локално стартиране / Run locally

```bash
npm install
npm start
# отвори http://localhost:3000/admin за качване
# отвори http://localhost:3000/?kiosk=1&rotate=20 за таблото
```

Задай администраторски ключ, за да защитиш админ страниците (`/admin.html`, `/messages.html`) и качването:

```bash
ADMIN_KEY=таен-ключ npm start
# отвори http://localhost:3000/login.html и влез с ключа
```

Без `ADMIN_KEY` админ страниците са отворени (удобно за локална работа). На публичен адрес **винаги задавай `ADMIN_KEY`**.

## URL параметри / URL params

| Параметър        | Действие                                              |
| ---------------- | ----------------------------------------------------- |
| `?kiosk=1`       | Скрива плаващите бутони (за постоянен дисплей)         |
| `?layout=board`  | Стартира с таблото (по подразбиране)                  |
| `?layout=grid`   | Стартира с цветната графика                           |
| `?rotate=20`     | Върти секциите (цехове) на всеки 20 секунди           |

**Препоръчан URL за дисплея:** `http://<адрес>/?kiosk=1&rotate=20`

## API

| Метод | Път              | Описание                                               |
| ----- | ---------------- | ------------------------------------------------------ |
| GET   | `/api/schedule`  | Връща разчетения график (JSON). 404 ако няма качен.    |
| GET   | `/api/status`    | Метаданни за polling (период, `parsedAt`, брой).       |
| POST  | `/api/upload`    | Качва `.xlsx` (multipart `file`). Header `x-admin-key`.|
| GET   | `/api/messages`     | Активни съобщения (публично, за таблото).           |
| GET   | `/api/messages/all` | Всички съобщения (admin, `x-admin-key`).            |
| POST  | `/api/messages`     | Ново съобщение (multipart: полета + по избор `image`). Admin. |
| DELETE| `/api/messages/:id` | Изтрива съобщение. Admin.                           |
| GET   | `/healthz`       | Health check.                                          |

**Админ страници (със защита):** `/admin.html` (качване на график) и `/messages.html` (управление на съобщения). Ако е зададен `ADMIN_KEY`, тези страници изискват **вход** през `/login.html` (подписана httpOnly сесия, валидна 12 часа). Самото табло (`/`) остава публично.

## Структура на .xlsx / Expected xlsx structure

Файлът има **две вертикални половини** които споделят редовете:
- **Ляво (колони A–G)** — жени: номер в A, двойки име/локация в B/C, D/E, F/G.
- **Дясно (колони H–N)** — мъже: номер в H, двойки в I/J, K/L, M/N.

Заглавия на смените: `1-ва смяна (07:00 – 15:00h)` и т.н. Цеховете (дясна половина): `цех НЕТЪКАН ТЕКСТИЛ`, `цех ХАРТИЯ И ПЛАСТМАСА`. Легендата с автобусните линии започва около ред 113 (име на линия, локации, брой).

Цветът на всеки работник се определя чрез съпоставяне на неговата локация с легендата (fill на клетката като резервен вариант) — така се запазва цветът дори при нова седмица.

## Deploy: Ubuntu 24.04 VM + Tailscale Funnel

**Една команда на новата VM:**

```bash
curl -fsSL https://raw.githubusercontent.com/SKuytov/septona-signage/master/install.sh | sudo bash
```

Инсталира Node.js 20 + PM2, клонира repo-то, стартира под PM2 (auto-start при boot) и публикува през Tailscale Funnel. Обновяване: `cd /opt/septona-signage && ./update.sh`.

Виж [`DEPLOY.md`](./DEPLOY.md) за опции (ADMIN_KEY, порт, без Tailscale), ръчна инсталация и troubleshooting.

## Android kiosk приложение / Android app

В [`android/`](./android/README.md) има пълен Android проект (WebView kiosk обвивка). Готовият `SeptonaSignage-debug.apk` се инсталира директно на дисплея. Компилиране от източник: `cd android && ./gradlew assembleDebug` (изисква JDK 17 + Android SDK 34).
